# Priority Classification Guide

Use this guide when manually reviewing or adjusting rule priorities.

## Alta — Apply strictly

Zero exceptions without explicit manager approval. Document all cases.

Typical examples:
- Legal deadlines ("refund must be issued within 7 calendar days")
- SLA breach penalties ("response time exceeded = 20% discount applied")
- Hard access cutoffs ("plan downgraded if payment not received within 3 days")
- Contractual obligations with financial consequences

Agent behavior: State the rule clearly. Do not negotiate. Escalate if customer pushes back.

## Media — Apply with judgment

Apply as the default. Deviations are allowed but must be documented in the ticket.

Typical examples:
- Standard response time targets ("first response within 4 business hours")
- Feature access tiers ("premium features available on Pro plan and above")
- Standard cancellation terms ("30-day notice required")

Agent behavior: Apply the rule. If customer has a legitimate edge case, note it and flag for supervisor review.

## Baixa — Apply as default

Supervisors can override without documentation. Exists to guide, not constrain.

Typical examples:
- Preferred communication channels ("email is preferred for billing questions")
- Soft recommendations ("customers are encouraged to use the self-service portal")
- Non-binding defaults

Agent behavior: Follow if applicable. No need to enforce rigidly.
